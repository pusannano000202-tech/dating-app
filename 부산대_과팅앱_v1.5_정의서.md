## 1. 프로젝트 한 줄 정의 (확정)

> **부산대에서 시작하는 대학생 그룹미팅 매칭 앱. 친구들과 그룹을 만들고 보증금을 걸면, 시스템이 상대 그룹·시간·장소를 자동 확정하는 "프로필 비공개 자동확정 과팅 앱".**

### 기존 데이팅앱과의 차별점

| 일반 데이팅앱 | Destiny |
|--------------|---------|
| 상대 프로필 보고 직접 선택 | 프로필 비공개 → 알고리즘이 매칭 |
| 채팅 후 만남 결정 | 채팅 없음 → 매칭 즉시 자동확정 |
| 1:1 중심 | 그룹미팅 우선 (2~5명) |
| 노쇼 방지 수단 없음 | 보증금 결제 → 출석자 환불, 노쇼 몰수 |
| 외모 자기 신고 | AI 사진 분석 + 월드컵 보정 |
| 무제한 리매칭 | 이전 만남 차단 (excluded_pairs) |

## 2. v1.1 → v1.4 주요 결정 변경 이력

### v1.1에서 바뀐 것 (원안 대비)
- ❌ Big5 AI 인터뷰(LLM 기반) → ✅ 고정 설문 (IPIP-NEO 단축형)
- ❌ 외모 타입 AI 단독 확정 → ✅ 이상형 월드컵 + AI 보정 혼합
- ❌ LLM API 사용 → ✅ 오픈소스 CV 모델(ResNet50 + SCUT-FBP5500)
- ➕ 보증금 시스템 추가 (핵심 차별화 기능)
- ➕ 절대 외모점수 내부 비공개 원칙 확립

### v1.2에서 바뀐 것 (v1.1 대비)
- ❌ 1:1 매칭 우선 → ✅ 그룹미팅 MVP 우선 (1:1은 후순위 확장)
- ❌ 학교 이메일 인증 → ✅ 휴대폰 OTP 인증 (학교정보는 프로필 입력값)
- ❌ 매칭 후 수락/거절 단계 → ✅ 신청+보증금 후 자동확정
- ❌ 매칭 전 상대 프로필 공개 → ✅ 만남 전까지 비공개
- ❌ 신고 1회 즉시 페널티 → ✅ 3회 이상 누적 후 신뢰도 반영
- ✅ 성비 보완 매칭 구체화 (남2↔여2, 남3↔여3, 혼성끼리 등)
- ✅ 헝가리안 알고리즘 기반 최적 그룹매칭 알고리즘 확정

### v1.4에서 바뀐 것 (v1.3 대비)
- ✅ 앱 이름 **Destiny** 공식 확정
- ✅ 브랜드 컨셉 "Cosmic Fate & Red Thread" 확정
- ✅ Manus 디자인 시스템 전면 반영
- ✅ Soul Orbs 매칭 풀 시각화 완성
- ✅ 이모지 전면 제거 → Lucide 아이콘 + 컬러 gradient 박스
- ✅ Playfair Display 브랜드 폰트 적용

## 4. 전체 서비스 플로우 (확정)

```
회원가입 (휴대폰 OTP)
    ↓
개인 프로필 작성 [충현 파트 — 완성]
  ① 이상형 월드컵 (8강 토너먼트)
  ② 자기유사 월드컵 (점수 비노출)
  ③ 기본 정보 (나이/키/학과/체형 등)
  ④ 사진 3장 업로드 → AI 점수 산출 (내부용)
  ⑤ Big5 성격 테스트
  ⑥ 가능 시간대 선택
  ⑦ 이상형 가중치 설정
    ↓
그룹 생성/초대 [성준 파트]
  - 2~5명 그룹
  - 초대 링크/코드
  - 전원 프로필 완성 필수
    ↓
보증금 결제 후 매칭 신청 [성준 파트]
  - 각 멤버 소액 보증금 결제
  - 신청 = 자동확정 동의
    ↓
주 1회 배치 그룹매칭 [성준 파트 — 헝가리안 알고리즘]
  - 동일 인원 + 성비 보완 그룹끼리만
  - 그룹 간 조화평균 점수 최대화
  - 개인 내부 불균형 페널티 적용
    ↓
자동확정 (인원/장소/시간 통보)
  - 만남 전 상대 사진/프로필 비공개
    ↓
만남 진행
    ↓
출석 인증 (GPS + 상호 인증)
  - 정상 출석 → 보증금 환불
  - 노쇼 → 보증금 몰수 → 출석자 보상
    ↓
만남 후 리뷰 + 호감 연결
  - 3회 이상 누적 리뷰만 신뢰도 반영
  - 상호 호감 → 1:1 채팅 또는 연락처 교환
```

## 5. 매칭 알고리즘 (확정)

### 5-1. 외모 점수 (내부용, 비공개)
- **모델**: SCUT-FBP5500 + ResNet50 (Python + PyTorch)
- **입력**: 업로드 사진 3장 평균
- **출력**: 0~100 절대점수 (DB 저장, 사용자에게 절대 노출 금지)
- **재평가**: 사진 교체 시 자동 재산출, 다음 매칭부터 반영

### 5-2. 외모 타입 (월드컵 + AI 보정)
- 이상형 월드컵: 사용자 선호 타입 선택 (선호 이미지 80% 반영)
- 자기유사 월드컵: 자기 이미지 선택 → AI 보정 40:60 혼합
- 타입 예시: 귀여움 / 청순 / 시크 / 훈훈 / 스타일리시 / 건강함

### 5-3. 그룹매칭 단계
1. **후보 필터**: 동일 인원수 + 성비 보완 + 가능 시간 교집합 + 보증금 완료 + excluded_pairs 제외
2. **개인 간 점수**: 각 멤버의 가중치 반영 쌍방향 점수 계산
3. **그룹 간 점수**: 조화평균 `2xy/(x+y)`
4. **내부 불균형 페널티**: 최저 개인 점수가 너무 낮으면 감점
5. **최적 배정**: 헝가리안 알고리즘으로 전체 만족도 최대화

### 5-4. 성격 점수 (Big5)
- 고정 설문 (IPIP-NEO 단축형 기반, 10문항 2개씩 5특성)
- 개방성/성실성/외향성/친화성/감수성 → 0~1 점수화
- 매칭 참고용. 사용자에게 결과 표시하되 상대에게 공개 안 함

### 5-5. 이상형 가중치
- 항목: 외모 / 성격 / 키 / 체형 / 학교 / 취미 / 시간대
- 슬라이더로 합계 100%가 되도록 비례 조정
- 그룹 평균 가중치도 매칭에 반영

## 6. 보증금 시스템 (확정)

| 단계 | 내용 |
|------|------|
| 신청 시 | 각 멤버 소액 보증금 결제 (초기 3,000원 권장) |
| 정상 출석 | 보증금 환불 |
| 노쇼 | 보증금 몰수 → 출석자 보상 재원 |
| N회 노쇼 | 매칭 N주 제한 → 반복 시 영구 제한 |

- **출석 검증**: GPS 체크인 + 상대 그룹 상호 인증
- **학교 프로젝트 단계**: 실제 결제 없이 DB 상태/PG 테스트로 플로우 구현
- **결제 상태값**: `paid` / `refunded` / `forfeited` / `compensated`

## 7. 리뷰/페널티 시스템 (확정)

- 만남 1회마다 상대방 리뷰 가능
- **3회 미만 리뷰**: 자동 페널티 없음
- **3회 이상 동일 항목 지적**: 프로필 신뢰도 하락
- 소수 의견이 다수와 크게 다르면 → 신고자 신뢰도 하락
- 리뷰 대상: 키/체형/사진 실물 차이/노쇼/지각/불쾌한 태도 (외모점수 자체 아님)
