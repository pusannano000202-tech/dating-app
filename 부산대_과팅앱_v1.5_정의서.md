## v1.5 핵심 변경 사항

### 1. self_appearance_score → self_appearance_vector (설계 확정)

**기존 (v1.4):** 자기유사 월드컵 결과를 단일 백분위 숫자로 저장
```
self_appearance_score: number  // 예: 68.0
```

**변경 (v1.5):** 6가지 외모 유형 차원의 벡터로 저장
```typescript
self_appearance_vector: {
  cute: number     // 귀여움 유사도 (0~100)
  pure: number     // 청순함 유사도 (0~100)
  chic: number     // 시크함 유사도 (0~100)
  warm: number     // 따뜻함 유사도 (0~100)
  stylish: number  // 스타일리시함 유사도 (0~100)
  healthy: number  // 건강미 유사도 (0~100)
}
```

**이유:**
- Codex가 각 자기유사 이미지를 생성할 때 6가지 외모 유형별 점수를 부여
- 단일 숫자보다 훨씬 정밀한 매칭 가능
- "내가 선호하는 외모 타입"과 "상대방이 실제로 보이는 외모 타입"을 직접 비교할 수 있음

**매칭 계산 방식:**
```
A의 appearance_type (이성 이상형 타입, 예: 'cute')
vs
B의 self_appearance_vector.cute (B가 귀여운 외모에 얼마나 가까운지)

→ 이 값의 높낮이가 외모 궁합 점수

양방향 계산:
  A→B: A가 선호하는 타입에 B가 얼마나 부합하는가
  B→A: B가 선호하는 타입에 A가 얼마나 부합하는가
```

**구현 변경 필요 사항 (PR 필요):**
- `lib/types.ts`: `self_appearance_score: number | null` → `self_appearance_vector: SelfAppearanceVector | null`
- `supabase/migrations/`: FLOAT 컬럼 → JSONB 컬럼으로 변경
- `app/profile/self-worldcup/page.tsx`: Supabase 저장 로직 변경
- ⚠️ Codex의 벡터 METADATA 완성 후 진행, 성준 리뷰 필수

### 2. 자기유사 이미지셋 128장으로 확장 (진행 중)

**기존:** 여자 32장 + 남자 32장 = 64장 (점수만 기록)

**변경:** 여자 64장(F01~F64) + 남자 64장(M01~M64) = 128장  
각 이미지에 점수 + 6차원 유형 벡터 기록

**기준 문서:** `public/appearance-self/DESIGN_128.md`  
**폴더 구조:**
```
public/appearance-self/
├── female-64/      F01.jpg ~ F64.jpg  (현재 16장 완료)
├── male-64/        M01.jpg ~ M64.jpg  (미시작)
├── female/         구 32장 (참고용 보존, 추가 작업 없음)
└── male/           구 32장 (참고용 보존, 추가 작업 없음)
```

**각 이미지의 METADATA 형식 (v1.5 신규):**
```
| ID  | 점수 | cute | pure | chic | warm | stylish | healthy |
|-----|------|------|------|------|------|---------|---------|
| F01 | 20   | 15   | 10   | 5    | 20   | 8       | 12      |
| F16 | 50   | 45   | 50   | 30   | 55   | 35      | 40      |
| F36 | 90   | 82   | 78   | 65   | 70   | 80      | 75      |
```

## 전체 시스템 아키텍처 (v1.5 기준)

```
사용자 → [이성 이상형 월드컵] → appearance_type 저장
         (어떤 외모 타입을 선호하는가: cute/pure/chic/warm/stylish/healthy)

사용자 → [자기유사 월드컵] → self_appearance_vector 저장
         (나는 어떤 외모 타입처럼 보이는가: 6차원 벡터)

매칭 엔진 →
  A.appearance_type 선호 강도 vs B.self_appearance_vector
  B.appearance_type 선호 강도 vs A.self_appearance_vector
  → 외모 궁합 점수 (양방향 평균)
  
  + Big5 성격 궁합
  + 시간대 겹침
  + 이상형 가중치(preference_weights)
  → 최종 매칭 점수
```
