## 1. 프로젝트 한 줄 정의

> **부산대에서 시작해 부산권 대학생으로 확장 가능한 그룹미팅 매칭 앱. 사용자가 친구들과 그룹을 만들고 보증금을 걸어 신청하면, 시스템이 상대 그룹·시간·장소를 자동으로 정해 실제 만남까지 성사시키는 "프로필 비공개 자동확정 과팅 앱".**

기존 데이팅앱과의 차별점:
- 프로필 쇼핑 X → 매칭 전후 상대 프로필/사진 비공개
- 채팅 단계 X → 매칭되면 바로 시간/장소 자동 확정
- 1:1보다 그룹미팅 우선 → 친구와 함께 신청해 접근성 상승
- 실제 소액 보증금 → 노쇼 방지와 서비스 신뢰 확보
- 외모 AI + 월드컵식 선택 → 사진 기반 객관 신호와 사용자 선호 신호를 함께 사용
- 만남 후 누적 리뷰 → 거짓말/노쇼/프로필 신뢰도 관리

## 4. 매칭 시스템 상세

### 4.1 프로필의 의미

프로필은 상대방에게 보여주는 소개장이 아니라, 매칭 엔진에 제출하는 비공개 데이터다.

사용자에게 공개하지 않는 정보:
- 절대 외모점수 (`appearance_score_normalized`)
- 자기유사 월드컵 백분위 점수 (`self_appearance_score`) — **사용자에게 절대 노출 금지**
- AI가 산출한 세부 점수
- 매칭 전 상대 사진/상세 프로필
- 매칭 점수 산식의 개별 결과

### 4.2 외모 점수

**1단 — AI 절대점수** `[구현됨 (프로토타입)]`
- 모델: ResNet50 (현재 ImageNet 사전학습 가중치 사용)
- `[미구현]` SCUT-FBP5500 파인튜닝 가중치 (`weights/resnet50_scut.pth`) 미적용 상태
- 사용자 사진 3장 평균 → 내부 0~1 정규화 점수 (`appearance_score_normalized`)
- 절대 외모점수는 사용자에게 공개하지 않는다.
- 사용자 화면에는 "사진 평가 완료", "다음 매칭부터 반영" 같은 상태만 표시

**사진 업데이트와 재평가**
- 사용자는 사진을 업데이트할 수 있다.
- 사진 업데이트 시 AI가 외모점수와 외모 타입을 재평가한다.
- 새 점수는 다음 매칭부터 반영한다.

### 4.3 외모 타입 + 자기유사 월드컵

`[수정됨]` v1.2에서 계획한 "자기 이미지 월드컵 40% + AI 60% 블렌딩" 방식은 1차 구현에서 채택하지 않음.

#### 4.3.1 이상형 월드컵 (선호 외모 타입)

- 사용자가 AI 생성 이미지 중 더 끌리는 쪽을 반복 선택
- 토너먼트 결과로 `appearance_type` 확정 (cute / pure / chic / warm / stylish / healthy)
- 사용자가 직접 선택한 취향이므로 AI 보정 없이 그대로 매칭 신호로 사용

#### 4.3.2 자기유사 월드컵 (내 외모 자가 평가) `[추가]`

`[수정됨]` v1.2에서 "자기 이미지 월드컵 결과는 그대로 확정값으로 쓰지 않고 AI 보정 대상"으로 계획했으나, 1차 구현에서는 사용자 선택 결과를 백분위 점수로 직접 저장. AI 보정은 추후 단계에서 검토.

**설계 원칙:**
- 프롬프트: "어느 쪽이 나와 더 비슷해 보여?"
- 이미지에 점수/등급 표시 없음 (사용자에게 점수 개념 노출 금지)
- Codex AI 생성 이미지 64장 (남/여 각 32장), 성별당 8장 계층적 샘플링 후 토너먼트 진행
- 이미지 파일: `/public/appearance-self/{gender}/{gender}_self_{score}.jpg`
- 결과: `self_appearance_score` — 0~100 백분위 스칼라 (DB 저장, 사용자에게 절대 노출 금지)

**점수 해석 기준:**
- 40점 = 같은 성별/연령대 전체 인구 중 하위 40%
- 90점 = 상위 10%
- 이 기준은 내부 알고리즘 전용이며 사용자 화면에 절대 표시하지 않는다.

**샘플링 전략 (성별당 8장):**

| 선택 점수 | 의미 |
|-----------|------|
| 20 | 하위 20% |
| 30 | 하위 30% |
| 40 | 하위 40% |
| 50 | 중간 |
| 60 | 상위 40% |
| 68/76/82 or 86 | 상위 24%/14%/10% |

- 1라운드 대진: 고점 vs 저점 시드 (86 vs 20, 76 vs 30 등) → 조기에 자기 수준대로 탈락/생존 유도
- 7경기로 단일 우승자 결정 (8강 토너먼트)

**매칭 활용 방향:**
- A의 `appearance_type` (이상형 외모 타입) ↔ B의 `self_appearance_score` (자기 백분위) — 양방향
- 예: "귀여운 타입 선호"인 A와 "귀여운 이미지 백분위 고점"인 B가 잘 매칭

#### 4.3.3 향후 AI 보정 계획 (미구현)

- 사진 업로드 후 AI가 외모 타입 후보 산출
- `self_appearance_score` (자기유사 월드컵) vs AI 분석 결과를 비교
- 큰 차이가 있을 때 보정값 반영 (비중 TBD)

### 4.4 성격 점수 `[구현됨]`

- 고정 설문 기반 (Big5 성격 테스트)
- 빅5 5차원 점수로 변환: openness, conscientiousness, extraversion, agreeableness, neuroticism
- LLM API 기반 분석 제외

### 4.5 가중치 시스템 `[구현됨]`

사용자가 매칭에서 중요한 요소를 직접 분배 (합계 = 1.0 보장):
- 외모, 성격, 키, 체형, 학교, 취미/분위기, 시간대 적합성

그룹매칭에서는 개인 가중치 + 그룹 평균 가중치 함께 계산.

### 4.6 그룹매칭 알고리즘 `[미구현 — 성준 담당]`

v1.2와 동일한 설계 유지:
- 동일 인원 그룹끼리만 매칭
- 성비 상호 보완 (남3↔여3, 남1여2↔남2여1 등)
- Step 1: 후보 필터링 → Step 2: 개인 간 점수 → Step 3: 그룹 간 조화평균 → Step 4: 불균형 페널티 → Step 5: 헝가리안 알고리즘
