# 부팅 디자인 리프레시 — 설계 스펙

날짜: 2026-06-10
상태: 사용자 승인 완료 (브레인스토밍 세션에서 비주얼 컴패니언으로 시안 확정)

## 목표

배포된 부팅 앱(gwating-app)의 사용자 페이지 전체를 "진짜 앱 느낌"이 나도록
일관된 디자인 시스템 위에 재구성한다. 디자인이 최우선이며, 코드 정리는
디자인 작업에 필요한 범위(토큰화, 공용 컴포넌트화)로 한정한다.

## 확정된 디자인 방향

브라우저 목업 비교를 통해 사용자가 직접 선택한 결과:

| 항목 | 결정 |
|------|------|
| 전체 무드 | A(비비드 그라디언트) 베이스 + C(클린 에디토리얼) 절제미 믹스 |
| 로고 | 그라디언트 볼트 — 코랄→오렌지 그라디언트 SVG 번개, 홈 중앙 히어로 배치 |
| 스플래시 | 파워온 부트업 — 번개 팝 → "부팅" 글자 바운스 → 로딩바, 세션당 1회 |
| 화면 전환 | 스태거 엔트런스 — 전환은 즉시, 페이지 요소가 위→아래 순차 등장 |
| 버튼 모양 | 16px 라운드 (알약 대신, 카드와 모서리 통일) |
| 앱 느낌 요소 | 터치 피드백(눌림 축소) + 데스크톱 모바일 프레임 |
| 페이지 범위 | 사용자 페이지 전부 9개, `/admin` 제외 |
| 애니메이션 구현 | 순수 CSS keyframes + Tailwind. 라이브러리 추가 없음 |

## 디자인 토큰 (`tailwind.config.ts` + `globals.css`)

- **컬러**
  - `primary` #FF4D67 / `primary-warm` #FF8A5C (그라디언트 쌍)
  - `ink` #191923 (본문), `muted` #8B8B94 (보조 텍스트)
  - `surface` #F7F8FA (카드/입력 배경), 캔버스는 #FFFFFF→#FFF0F1 그라디언트
  - 그라디언트는 named 유틸(`bg-boot-gradient` 등)로 등록 — 페이지별 hex 복붙 금지
- **모서리**: 카드 16px / 버튼 16px / 칩 12px
- **그림자**: `shadow-glow`(코랄 컬러 섀도우, primary CTA 전용), `shadow-card`(중립) 2종만 사용
- **폰트**: Pretendard Variable 유지
- **모션 유틸**: 스태거 키프레임 + 자식 순차 delay 유틸(0.06s 간격).
  `prefers-reduced-motion: reduce` 시 전부 비활성화

## 데스크톱 모바일 프레임 (`app/layout.tsx`)

- 모바일(<480px): 현재처럼 풀스크린
- 데스크톱: 회색 배경 중앙에 420px 폭 폰 프레임(라운드 + 그림자), 앱은 프레임
  내부에서 렌더링되고 스크롤도 프레임 내부로 한정
- 기존 페이지의 `min-h-screen`은 프레임 안에서 깨지므로 PageShell이 흡수

## 공용 프리미티브 (`components/ui/`)

| 컴포넌트 | 역할 |
|----------|------|
| `PageShell` | 페이지 최상위 래퍼. 캔버스 배경 + 공통 패딩 + 스태거 엔트런스 자동 적용. 프레임 내 높이 처리 담당 |
| `Button` | 기존 Button 교체. `primary`(그라디언트+글로우)/`secondary`(흰 배경+테두리)/`ghost`. 터치 피드백 내장(active 시 scale 0.97 + 그림자 가라앉음) |
| `Card` | 16px 라운드 + `shadow-card` 서피스. `pressable` 옵션 시 터치 피드백 |
| `Chip` | 선택 칩(12px), 선택 시 코랄 하이라이트. 기존 `MoodChip` 흡수 |
| `BoltLogo` | 그라디언트 볼트 SVG 컴포넌트(크기 prop). 헤더/스플래시/홈 히어로 재사용 |
| `Splash` | 파워온 부트업 연출(~1.8초) 후 자동 해제. sessionStorage로 세션당 1회 |

기존 도메인 컴포넌트(TeamProfileCard, MatchScoreCard, QuizCard,
RecommendationTeamCard, AppHeader 등)는 삭제하지 않고 프리미티브/토큰 위에서
리스타일한다.

## 페이지별 적용

모든 페이지를 `PageShell` 위에 올린다.

- **홈 `/`**: 볼트 히어로 중앙(글로우 펄스 + 플로팅), 헤드라인 그라디언트 텍스트
  포인트, 장식 이모지(🎉✨💬) 제거, 첫 진입 시 스플래시
- **성향 테스트 `/test`**: 그라디언트 진행바, 질문은 `Card`, 답변은 pressable
  카드. 질문 전환마다 스태거 재생
- **팀 생성 `/team/create`**: 폼/무드 선택을 `Chip`·토큰으로 통일, 위계 정리
- **팀 프로필 `/team/demo`**: TeamProfileCard 리스타일
- **매칭 결과 `/match/result`**: 스코어 링 그라디언트화, 카드 리스타일
- **일정/확정/Q&A `/match/schedule·confirmed·qa`**: 토큰·컴포넌트 일괄 적용
  (구조 변경 없음)
- **채팅 `/match/chat`**: 내 말풍선 코랄 그라디언트, 상대 말풍선 surface,
  입력바 정돈

**불변 조건**: 로직(상태 관리, Supabase 연동, 매칭/스코어 계산, 라우팅 흐름)은
변경하지 않는다. 스타일과 마크업만 교체한다.

## 작업 순서

1. 파운데이션 — 토큰, 모바일 프레임, 스태거 유틸
2. 프리미티브 — BoltLogo, Button, Card, Chip, PageShell, Splash
3. 홈 + 스플래시 (룩 쇼케이스 — 여기서 사용자 컨펌 후 진행)
4. 성향 테스트 → 팀 생성/프로필 → 매칭 플로우 5개 순서로 마이그레이션
5. 미사용 구 스타일 코드 정리

## 검증

- 단계마다 `npm test` + `npm run build` 통과 (로직 무변경의 회귀 감지선)
- 단계마다 dev 서버로 시각 확인, 특히 3단계 후 홈 룩 사용자 컨펌
- `prefers-reduced-motion` 동작 확인
- 구현 완료 후 Codex 감사(`/codex:review`) → Critical 0까지 반영 (CLAUDE.md 헌법)
