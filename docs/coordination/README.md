# 팀장방 운영 허브

이 폴더는 팀장방과 하위 직원이 같은 기준으로 일하기 위한 고정 문서 모음이다.

중요한 원칙은 하나다. 기억은 대화방 머릿속에 두지 않고, repo 안의 문서로 공유한다.
하위 직원은 필요할 때 새로 만들 수 있지만, 역할명, 권한, 시작 문서, 보고 형식은 고정한다.

## 문서 목록

| 문서 | 목적 |
| --- | --- |
| `TEAM_LEAD_RUNBOOK.md` | 팀장방이 작업을 분류하고 배정하는 운영 절차 |
| `AGENT_ROLES.md` | 고정 직원 역할표, 권한, 금지사항 |
| `ROLE_PROMPTS.md` | 하위 직원에게 그대로 붙여 넣는 역할별 시작 프롬프트 |
| `TEAM_CONTEXT.md` | 제품 방향, 현재 결정사항, 주요 경로, 금지사항 |
| `WORKER_REPORT_TEMPLATE.md` | 직원이 작업 완료 후 남기는 보고 형식 |
| `ACTIVE_ASSIGNMENTS.md` | 현재 진행 중인 작업, 완료 작업, 커밋 전 묶음 추적 |

## 기본 운영 흐름

1. 사용자는 팀장방에만 최종 명령을 내린다.
2. 팀장방은 작업을 구조 감사, UX 감사, DB/API 검토, 프론트 구현, 에셋 작업으로 분류한다.
3. 팀장방은 필요한 고정 역할만 부른다.
4. 하위 직원은 지정된 범위 밖을 건드리지 않는다.
5. 하위 직원은 커밋하지 않는다.
6. 하위 직원은 결과를 보고 형식으로 남긴다.
7. 팀장방이 결과를 취합하고, 충돌을 판단하고, 검증하고, 커밋 여부를 보고한다.

## 하위 직원에게 기본으로 읽히는 문서

하위 직원을 만들 때 최소한 아래 문서를 읽게 한다.

```text
docs/coordination/AGENT_ROLES.md
docs/coordination/TEAM_CONTEXT.md
docs/coordination/WORKER_REPORT_TEMPLATE.md
```

구현 작업이면 추가로 해당 작업 지시서, 관련 파일 목록, 수정 허용 범위를 같이 전달한다.

## 이름 규칙

도구가 자동으로 `Ptolemy`, `Peirce` 같은 별칭을 줄 수 있다.
그래도 팀장방 보고에서는 아래 역할명을 우선 사용한다.

```text
Noether = 구조/흐름 감사
McClintock = UX/디자인 감사
Bernoulli = DB/API 검토
Darwin = 프론트 구현
Pascal = 이미지/에셋
```

자동 별칭과 역할명이 다르면 `ACTIVE_ASSIGNMENTS.md`에 매핑을 남긴다.
