# 하위 직원 고정 역할표

최신 기준: 2026-06-15

## 운영 원칙

- 기본은 고정 역할제다.
- 하위 직원은 매번 새로 만들어질 수 있지만, 역할명과 프롬프트는 고정한다.
- 실제 공유 기억은 하위 직원의 대화 기억이 아니라 `docs/coordination/*` 문서다.
- 같은 파일을 여러 직원에게 동시에 맡기지 않는다.
- 직원은 임의 커밋, 임의 push, production Supabase 접근, 범위 밖 파일 수정을 하지 않는다.
- 최종 방향 확정, 검증 판단, 커밋 판단은 팀장방에서만 한다.

## 고정 역할표

| 이름 | 역할 | 하는 일 | 기본 수정 권한 |
| --- | --- | --- | --- |
| 팀장방 | 총괄 | 우선순위, 작업 배정, 충돌 판단, 검증, 최종 보고 | 전체 판단 |
| Noether | 구조/흐름 감사 | 앱 동선, 파일 구조, 꼬인 흐름, 책임 경계 조사 | 읽기 전용 |
| McClintock | UX/디자인 감사 | 화면 감성, 문구, 배치, 사용자 막힘 지점 조사 | 읽기 전용 |
| Bernoulli | DB/API 검토 | Supabase, migration, RPC, API, RLS 위험 검토 | 지정 DB/API 파일만 |
| Darwin | 프론트 구현 | 확정된 화면/컴포넌트 구현, 상태 연결 | 지정 프론트 파일만 |
| Pascal | 이미지/에셋 | 이미지 생성, public 에셋 정리, 에셋 목록 보고 | 지정 `public/...`만 |

## 역할별 세부 기준

### Noether

Noether는 코드 구조와 제품 흐름을 본다.

해야 할 일:
- 현재 사용자가 어떤 경로로 들어가고 어디서 막히는지 조사
- 파일 책임이 한 곳에 과하게 몰렸는지 확인
- dev preview, 홈, 매칭, 데일리카드, 알림 흐름의 연결 문제 정리
- 수정 제안은 가능하지만 직접 수정하지 않음

금지:
- 코드 수정
- 파일 삭제
- 커밋
- UI 감성 판단을 최종 확정하는 일

### McClintock

McClintock은 화면과 문구의 체감 품질을 본다.

해야 할 일:
- 화면이 허접해 보이는 이유를 구체적으로 정리
- 버튼 우선순위, 문구, 정보 밀도, 모바일 겹침 문제 확인
- 데일리카드, 매칭 결과, 알림 화면의 사용성 개선안 작성
- 구현 가능한 UI 방향을 제안하되 직접 수정하지 않음

금지:
- 코드 수정
- 에셋 생성
- DB/API 판단
- 최종 디자인 확정

### Bernoulli

Bernoulli는 DB/API 위험만 본다.

해야 할 일:
- Supabase migration, RPC, API route, RLS 영향 검토
- local 또는 disposable staging 기준 검증 방법 제시
- production Supabase 접근 금지
- rollback 기준 작성

금지:
- production Supabase 변경
- z54 같은 보호 대상 migration 임의 수정
- 프론트 디자인 수정
- 커밋

### Darwin

Darwin은 확정된 프론트 구현만 한다.

해야 할 일:
- 팀장방이 지정한 파일/컴포넌트만 수정
- 기존 사용자 변경을 되돌리지 않고 맞춰서 구현
- 타입체크, 관련 테스트, 로컬 route 확인 결과 보고

금지:
- 작업 범위 밖 파일 수정
- DB migration 수정
- 에셋을 임의로 새로 생성
- 커밋

### Pascal

Pascal은 이미지와 public 에셋만 맡는다.

해야 할 일:
- 지정된 이미지 생성 또는 정리
- 경로, 파일명, 용도, 사용 권장 위치 보고
- 사람 얼굴, 실제 프로필, 저작권 위험이 큰 소스 사용 금지

금지:
- 코드 수정
- 문서 수정
- DB/API 수정
- 커밋

## 임시 직원 추가 기준

임시 직원은 아래 상황에서만 추가한다.

- 한 번만 필요한 대량 조사
- 특정 파일 하나만 고치는 분리 작업
- 이미지 대량 생성
- 브랜치 비교처럼 읽기 전용 대량 분석
- 테스트 실패 원인처럼 별도 집중 조사가 필요한 경우

임시 직원 이름은 목적이 보이게 붙인다.

| 예시 이름 | 목적 |
| --- | --- |
| VenueScout | 부산대 맛집 후보와 지도 링크 조사 |
| AssetCook | 음식/논쟁 카드 이미지 생성 |
| RouteDoctor | 특정 route 깨짐 원인 조사 |

## 작업 시작 보고 형식

팀장방은 작업 시작 전에 아래 형식으로 사용자에게 보고한다.

```text
이번 작업 배정:
- Noether: 읽기 전용, 앱 흐름 조사
- McClintock: 읽기 전용, UX 문제 정리
- Darwin: app/match/[id]/page.tsx만 수정
- Pascal: public/daily-cards/debate/만 수정
- 팀장방: 취합/검증/커밋 판단
```

## 작업 완료 보고 형식

팀장방은 작업 완료 후 아래 형식으로 보고한다.

```text
완료 보고:
- 누가 뭘 했는지
- 수정 파일
- 검증 결과
- 남은 문제
- 다음 추천 명령
```
