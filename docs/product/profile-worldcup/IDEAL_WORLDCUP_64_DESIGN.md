# 이상형 월드컵 64강 이미지 설계서

이 문서는 자기유사 월드컵이 아니라 **상대방 이상형 월드컵** 전용 이미지 설계서다.

사용자는 본인과 비슷한 사람을 고르지 않는다. 사용자는 이성 이미지 64장 중에서 끌리는 쪽을 고른다. 결과는 `preferred_appearance_vector`로 저장한다.

중요:

- 이 설계표의 유형/점수는 이미지 생성용 목표값이다.
- 최종 매칭에는 생성 의도인 `target_type`을 직접 쓰지 않는다.
- 생성된 이미지는 GPT 분석기로 다시 평가해서 `measured_vector`를 만들고, 실제 월드컵 선호 계산에는 그 값을 사용한다.
- 자세한 운영 기준은 `docs/IDEAL_WORLDCUP_MEASURED_VECTOR_PLAN.md`를 따른다.

## 1. 전체 구조

### 본인 외모 분석

```text
사용자 사진 업로드
→ AI가 appearance_score_normalized와 appearance_vector 추출
```

나오는 값:

- 내 외모 점수
- 내 외모 타입/벡터

### 상대방 이상형 월드컵

```text
이성 이미지 64장 중 선택
→ measured_vector 기반 preferred_appearance_vector 추출
```

나오는 값:

- 내가 선호하는 외모 벡터
- 선호 상위 유형
- 선호 점수대
- 선택 로그

### 매칭 반영

```text
A의 선호 벡터 vs B의 실제 외모 벡터
B의 선호 벡터 vs A의 실제 외모 벡터
```

양방향 유사도를 외모 궁합 점수로 사용한다.

## 2. 이미지 수량

성별별로 이상형 풀을 따로 만든다.

```text
남자가 보는 여자 이상형 이미지: 64장
여자가 보는 남자 이상형 이미지: 64장
총 128장
```

권장 파일 위치:

```text
public/appearance-ideal/female-64/FI01.jpg ~ FI64.jpg
public/appearance-ideal/male-64/MI01.jpg ~ MI64.jpg
```

## 3. 점수 분포

이상형 월드컵은 자기유사 월드컵과 다르다. 자기유사용은 20점대부터 넓게 깔지만, 이상형 월드컵은 사용자가 끌리는 이미지를 고르는 것이므로 전부 어느 정도 호감형이어야 한다.

64장 권장 분포:

| 점수대 | 수량 | 역할 |
|---|---:|---|
| 60점대 | 12장 | 현실적 호감형, 취향 하한선 |
| 70점대 | 28장 | 주력 취향 분화 구간 |
| 80점대 | 20장 | 강한 선호를 끌어내는 구간 |
| 90점대 | 4장 | 최상위 기준점, 과다 사용 금지 |

점수는 사용자에게 보여주지 않는다. 내부 벡터 계산과 이미지 풀 균형 검수에만 사용한다.

## 4. 공통 생성 프롬프트

각 행의 핵심 프롬프트 앞뒤에 아래 공통 조건을 붙인다.

```text
완전히 가상의 20대 초반 한국 대학생 인물.
실존 인물, 연예인, 유명인과 닮지 않음.
상대방 이상형 월드컵용 기준 이미지.
세로형 3:4 비율, 가슴 위 상반신 프로필 카드 구도.
깨끗하고 단순한 배경, 자연스러운 조명, 실제 프로필 사진 느낌.
과한 노출 없음, 선정적이지 않음.
글자, 로고, 워터마크 없음.
피부는 점수대와 유형에 맞게 현실감을 유지하고, 모든 이미지를 똑같은 인형 피부로 만들지 않음.
같은 유형 안에서도 헤어, 옷, 표정, 얼굴형이 반복되지 않게 함.
AI 생성 이미지처럼 보이지 않게 현실적인 비대칭, 자연스러운 피부 질감, 작은 잡티, 미세한 잔머리, 옷 주름, 평범한 카메라 거리감을 유지함.
눈동자 하이라이트, 치아, 손, 귀, 머리카락 경계, 안경테가 과하게 매끈하거나 비현실적으로 깨끗하지 않게 함.
얼굴 좌우가 완벽하게 대칭이거나, 피부가 유리처럼 매끈하거나, 조명이 광고 화보처럼 과장되면 실패로 봄.
실제 사용자가 데이팅 앱에서 볼 법한 자연스러운 프로필 사진이어야 하며, 스튜디오 화보나 연예인 프로필 사진처럼 보이면 안 됨.
```

낮은 60점대는 "못생김"이 아니라 현실적 호감형으로 만든다. 90점대는 소수 기준점이므로 과하게 많이 만들지 않는다.

### AI 티 방지 공통 규칙

이상형 월드컵 이미지는 사용자가 실제 사람처럼 받아들일 수 있어야 한다. 너무 완벽한 AI 미남/미녀 사진은 거부감을 만들고, 사용자의 선택도 현실 취향이 아니라 "AI가 만든 예쁜 얼굴" 쪽으로 왜곡된다.

프롬프트에는 아래 현실감 조건을 공통으로 넣는다.

- 피부는 현실적인 질감을 가진다. 모공, 작은 잡티, 약한 피부톤 차이, 자연스러운 그림자는 허용한다.
- 얼굴 좌우 대칭을 완벽하게 만들지 않는다. 눈 크기, 입꼬리, 볼륨, 머리카락 흐름에 미세한 차이가 있어야 한다.
- 눈동자 하이라이트가 지나치게 반짝이거나 유리알처럼 보이면 안 된다.
- 치아가 보이는 경우 완벽한 치열/과한 백색 치아를 피한다.
- 귀 모양과 귀 위치가 매번 다르게 보여야 하며, 귀가 보이는 슬롯에서는 귀가 실제 사람처럼 자연스럽게 보여야 한다.
- 머리카락 경계가 헬멧처럼 매끈하면 안 된다. 잔머리, 삐져나온 머리, 자연스러운 머리카락 흐름을 넣는다.
- 옷은 새 상품 화보처럼 완벽하면 안 된다. 약한 주름, 자연스러운 핏, 생활감이 있어야 한다.
- 배경은 과하게 고급스럽거나 블러가 심한 화보 배경보다, 학교/카페/도서관/실내/야외 같은 현실적인 장소감을 우선한다.
- 카메라는 과도한 뷰티 필터, 과한 보케, 과한 색보정, 과한 샤픈을 피한다.
- 60~70점대는 특히 현실 프로필 사진처럼 보여야 한다. AI가 만든 인형 같은 얼굴이면 실패다.

생성 프롬프트에 넣을 수 있는 문장 예시:

```text
Make it look like a realistic dating-app profile photo taken with a normal phone camera, not an AI glamour portrait. Keep subtle facial asymmetry, real skin texture, tiny skin imperfections, natural hair flyaways, ordinary fabric wrinkles, and believable indoor or outdoor lighting. Avoid plastic-smooth skin, perfectly symmetrical features, glassy oversized eyes, over-white teeth, overly cinematic lighting, fashion editorial styling, and celebrity-like polish.
```

## 5. 중복 방지 강제 차별화 설계

아래 표는 이미지 생성 전에 반드시 각 행의 핵심 프롬프트에 추가한다. 목표는 128장 중 최소 95%가 서로 다른 사람처럼 보이게 만드는 것이다.

원칙:

- 같은 대분류 안에서도 얼굴형, 눈매, 헤어, 옷, 표정, 배경 중 최소 4개 이상이 달라야 한다.
- 여자 이미지는 긴 생머리, 흰 옷, 자연스러운 미소로 몰리지 않게 한다.
- 남자 이미지는 짧은 검은 머리, 셔츠, 무표정으로 몰리지 않게 한다.
- 60점대도 호감형이어야 하지만 피부, 얼굴 비율, 스타일을 너무 완벽하게 만들지 않는다.
- 80~90점대는 얼굴 조화가 좋아야 하지만 실존 연예인 느낌이나 같은 AI 미남/미녀 얼굴로 수렴하면 실패로 본다.
- 생성 후 접촉 시트에서 20장 중 1장 이상이 "같은 사람의 다른 컷"처럼 보이면 해당 묶음을 재생성한다.

### 프롬프트 재정비 원칙

초기 생성 검수 결과, `귀여움`, `청순함`, `차분함`, `상위권`, `호감형` 같은 분위기 단어만으로는 이미지 생성기가 비슷한 한국 AI 얼굴로 수렴했다. 따라서 이후 생성 프롬프트는 아래 구조를 반드시 따른다.

```text
공통 생성 프롬프트
+ 점수/대분류/세부 분류
+ 기존 핵심 프롬프트
+ 차별화 키(얼굴형/눈매/헤어/옷/표정/배경)
+ 정밀 얼굴 부품 키(눈/눈썹/코/입/귀/헤어색/목선/어깨선/네크라인)
```

숫자형 설계는 내부 기준으로만 사용한다. 실제 이미지 생성 프롬프트에는 숫자를 그대로 쓰지 않고 자연어로 번역한다.

예:

```text
눈 크기 4/5, 눈꼬리 -1, 얇은 속쌍, 눈 사이 거리 3/5
→ 큰 편의 둥근 눈, 눈꼬리가 살짝 내려간 순한 눈매, 얇은 속쌍, 눈 사이가 보통보다 약간 넓은 인상
```

선정적으로 보일 수 있는 신체 항목은 직접 수치화하지 않는다. `가슴 크기` 대신 `상반신 실루엣`, `어깨선`, `목선`, `쇄골 노출 정도`, `옷 핏`, `네크라인`으로만 관리한다.

현재 생성된 `FI01~FI18`은 1차 초안으로 본다. 최종 이미지셋을 만들 때는 아래 정밀 설계표를 적용해 재생성하거나, 접촉 시트 기준으로 통과한 이미지만 유지한다.

## 10. 64강 선택 결과 계산

64강 풀모드 기준 선택 수:

```text
64강 → 32강 → 16강 → 8강 → 4강 → 결승
총 63회 선택
```

선택 가중치:

| 라운드 | 가중치 |
|---|---:|
| 64강 | 1.0 |
| 32강 | 1.2 |
| 16강 | 1.5 |
| 8강 | 2.0 |
| 4강 | 2.5 |
| 결승 | 3.0 |

최종:

```text
preferred_appearance_vector
= 선택된 이미지 벡터의 가중 평균
```

우승자 하나만 저장하지 말고, 전체 선택 로그를 저장해야 한다.

저장 권장값:

```json
{
  "preferred_appearance_vector": {},
  "preferred_top_types": ["귀여운/동안형", "청순/자연형"],
  "preferred_score_band": "70~80점대",
  "selection_log": []
}
```

## 11. 검수 기준

생성 후 반드시 아래를 확인한다.

- 접촉 시트 기준으로 128장 중 최소 122장 이상이 서로 다른 사람처럼 보여야 한다. 122장 미만이면 95% 미달로 보고 재설계한다.
- 같은 사람의 헤어/옷/배경만 바꾼 것처럼 보이는 이미지는 중복 실패로 본다.
- 눈, 코, 입, 귀가 비슷한 평균형으로 반복되면 헤어/옷/배경이 달라도 중복 실패로 본다.
- 귀가 계속 머리카락에 가려져 있거나 전부 같은 귀 모양으로 보이면 중복 실패로 본다.
- 헤어색이 전부 자연 흑발로 몰리면 실패다. 흑발 중심은 유지하되 다크브라운, 톤다운 브라운, 라이트브라운, 애쉬브라운이 일부 섞여야 한다.
- 상반신은 선정적이지 않아야 하며, 목선/어깨선/네크라인/옷 핏 차이가 사진에서 보여야 한다.
- AI 생성 이미지 특유의 매끈한 피부, 과한 대칭, 유리알 같은 눈동자, 과한 치아 보정, 지나치게 완벽한 머리카락 경계가 보이면 실패로 본다.
- 60~70점대 이미지가 화보처럼 과하게 예쁘거나, 80~90점대 이미지가 연예인 프로필처럼 보이면 실패로 본다.
- 접촉 시트를 봤을 때 "실제 데이팅 앱에 올라온 자연스러운 프로필 사진"이 아니라 "AI가 만든 미남/미녀 모음"처럼 보이면 재생성한다.
- 각 8장 묶음 안에서 얼굴형, 눈매, 헤어, 옷, 표정, 배경 중 최소 4개 이상이 반복 없이 갈라져야 한다.
- 같은 유형 8장이 서로 너무 비슷하지 않은가?
- 60점대가 너무 낮거나 재미없게 보이지 않는가?
- 90점대가 너무 많아 보이지 않는가?
- 여자 이미지가 전부 긴 생머리/흰 옷으로 몰리지 않았는가?
- 남자 이미지가 전부 짧은 검은 머리/무표정/셔츠로 몰리지 않았는가?
- 유형별 차이가 사진만 보고도 느껴지는가?
- 사용자 화면에 점수/하위/상위 표현이 노출되지 않는가?

95% 미달일 때의 보강 순서:

1. 중복으로 보이는 ID를 먼저 표시한다.
2. 같은 대분류 안에서 겹치는 얼굴형/헤어/옷/표정/배경을 찾는다.
3. 눈/코/입/귀/헤어색/상반신 키가 실제로 다른지 확인한다.
4. 중복 ID의 차별화 키와 정밀 얼굴 부품 키를 먼저 바꾼다.
5. 그래도 부족하면 핵심 프롬프트의 분위기 단어를 줄이고 물리적 특징을 늘린다.
6. 재생성 후 접촉 시트를 다시 만들어 122장 이상이 서로 다른 사람처럼 보이는지 확인한다.

## 12. 생성 순서

1. 각 ID별로 공통 프롬프트, 기존 설계표, 차별화 키, 정밀 얼굴 부품 키를 합쳐 최종 생성 프롬프트를 만든다.
2. 여자 64장을 새 정밀 프롬프트 기준으로 생성한다.
3. 여자 64장 접촉 시트를 만든다.
4. 여자 이미지의 얼굴 부품 중복, 점수 감각, 유형 불일치를 검수한다.
5. 여자 이미지 교체 후보를 정하고 재생성한다.
6. 여자 64장을 확정한다.
7. 남자 64장도 같은 방식으로 진행한다.
8. 전체 128장 접촉 시트를 만든다.
9. 전체 128장 기준으로 95% 다양성 기준을 검수한다.
10. 각 이미지 벡터 JSON과 메타데이터를 작성한다.
11. 월드컵 UI에서 이미지 풀을 연결한다.
12. 선택 로그 기반 `preferred_appearance_vector` 계산을 구현한다.
