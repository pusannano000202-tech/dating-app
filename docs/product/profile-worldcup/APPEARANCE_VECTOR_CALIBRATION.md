# 외모 분석 점수·벡터 보정 기준표

이 문서는 GPT Vision 분석기가 일관된 점수와 벡터를 내도록 하기 위한 보정 기준이다.
프롬프트 본문은 `docs/APPEARANCE_ANALYSIS_GPT_PROMPT.md`, 출력 구조는 `docs/APPEARANCE_ANALYSIS_SCHEMA.md` 참조.

이 문서의 기준은 사용자에게 직접 공개되지 않는다. 매칭 알고리즘 내부 보정에만 쓴다.

---

## 0. 단일 점수 정의 출처 (Single Source of Truth)

**외모절대점수의 정의는 `public/appearance-self/SCORE_GUIDE.md` 가 유일한 출처다.**

이 문서, GPT 프롬프트, JSON 스키마, 회귀 테스트는 전부 SCORE_GUIDE.md 를 따른다.
충돌하는 부분이 발견되면 SCORE_GUIDE.md 가 이긴다.

SCORE_GUIDE.md 의 핵심 정의를 다시 옮긴다.

> 외모절대점수는 호감도 느낌 점수가 아니라 **백분위 점수**다.
>
> - 90점 = 같은 성별·비슷한 연령대 100명 중 **상위 10명** 수준
> - 70점 = 100명 중 **상위 30명** 수준
> - 50점 = 평균 근처
> - 30점 = 100명 중 **하위 30명** 수준
> - 10점 = 100명 중 **하위 10명** 수준
>
> 40점은 "나쁘지 않다"가 아니라 외모 분포에서 **하위 40퍼센트 지점**이라는 뜻이다.

즉 점수 = 백분위 위치(percentile rank). 선형 분포. 정규분포가 아니다.

자기유사 월드컵 이미지 64장(`public/appearance-self/`)은 이 정의로 만들어졌다. 이상형 월드컵 이미지 64장(`public/appearance-ideal/`)도 60~94점 = 상위 6~40% 구간을 커버한다. 사용자 사진 분석 결과도 같은 정의로 매겨져야 매칭에서 비교가 성립한다.

## 1. 점수 보정 기준 (핵심)

### 1-1. 점수대 정의 (백분위 기준 — Codex SCORE_GUIDE.md 그대로)

| 점수 | 백분위 | 모집단 위치 |
|---:|---|---|
| 90~100 | 상위 0~10% | 매우 뛰어난 외모. 대부분이 미남/미녀로 인식 |
| 80~89 | 상위 10~20% | 확실히 눈에 띄는 미남/미녀 |
| 70~79 | 상위 20~30% | 대체로 예쁘다/잘생겼다고 평가받는 호감형 |
| 60~69 | 상위 30~40% | 평균 이상. 호감 요소가 분명함 |
| 50~59 | 상위 40~50% | 평균 근처 |
| 40~49 | 하위 40~50% | 평균 이하. 미형 신호가 약함 |
| 30~39 | 하위 30~40% | 낮은 호감도 구간. 불리한 외모/관리 요소가 보임 |
| 20~29 | 하위 20~30% | 뚜렷하게 불리한 구간 |
| 10~19 | 하위 10~20% | 극단적 하위 구간 |
| 0~9 | 하위 0~10% | 매우 극단적 구간. 실서비스에서는 거의 부여되지 않음 |

핵심: 60점은 상위 40% 위치다. "현실적 호감형" 이라는 표현은 60점이 평균 이상이라는 의미일 뿐, "그냥 무난한 사람" 이 아니다.

### 1-2. 점수 부여 절차

GPT 가 점수를 매기는 순서:

1. 사진의 photo_quality 부터 평가한다. 불량이면 점수 산출을 보수적으로 한다(50 부근 + 낮은 score_confidence).
2. 보정/조명/카메라 보정 효과를 제외한 "있는 그대로" 의 인상을 추정한다.
3. "이 사람이 같은 성별·연령대 100명 중에서 위에서부터 몇 번째 자리에 들겠는가?" 를 자문한다.
4. 그 자리 번호의 반대편 백분위 = 점수. (예: 위에서 30번째 → 70점, 위에서 70번째 → 30점)
5. 같은 구간 안에서 헤어/스타일/표정 정돈감을 보고 ±5 미세 조정.

### 1-3. 하위권 과대평가 방지 (가장 중요)

LLM 은 사람의 외모를 깎아내리는 답을 회피하는 경향이 강하다. 그 결과 하위 20~40% 외모(20~40점)도 60~70점대로 부풀리는 편향이 흔히 발생한다. 이 시스템에서는 그 편향이 매칭 품질을 직접 파괴한다.

**왜 정직해야 하는가**

- 이 점수는 사용자에게 노출되지 않는다.
- 점수가 부풀려지면 하위권 사용자가 상위권 사용자와 매칭되어 양쪽 모두 만족하지 못한다.
- 비슷한 백분위끼리의 매칭이 양쪽 만족도가 가장 높다. 정직한 점수가 결국 사용자 보호다.

**구체적 가드레일**

1. "이 점수를 매기는 게 사용자에게 상처가 될까?" 하는 망설임이 생기면, 그 망설임 자체가 과대평가 신호다. 점수를 더 낮춰라.
2. 사진이 호의적으로 잘 찍힌 경우(좋은 각도, 좋은 조명, 약간의 보정)는 그 효과를 빼고 백분위를 다시 추정해 5~10점 낮춘다.
3. heavy_filter_suspected=true 일 때는 보정 효과를 제거한 추정으로 점수를 다시 매긴다. 보정 흔적이 강하면 추가로 5~10점 더 깎는다.
4. 한 배치(예: 한 시간 안에 분석된 50명) 에서 60점 초과 비율이 50% 를 넘는다면, 그 배치는 과대평가 편향이 발생한 것이다. 정의상 60점 초과는 상위 40% 이내인데 일반 모집단의 절반 이상이 거기에 들 수 없다. 운영팀이 회귀 테스트(섹션 5)를 즉시 돌려야 한다.
5. 30~40점대 점수를 부여하기 전에 "이 인상의 사람이 100명 중 위에서 60~70번째 자리에 들겠는가?" 만 자문한다. "이 사람 기분이 어떨까" 는 자문하지 않는다.
6. "현실적 호감형" 이라는 표현(60~69점)은 상위 30~40% 라는 뜻이다. 모집단 절반 이상이 60점에 못 미친다. 60점 이상을 남발하지 마라.

**상한 가드레일도 둔다**

- 90점 이상은 상위 10% 라는 정의다. 일반 모집단에서 10명 중 1명만 이 점수를 받는다.
- 95점 이상은 상위 5% 다. 100명 중 5명. 신중히 부여.
- 80점대(상위 10~20%) 도 매우 흔히 부여하면 안 된다. 5명 중 1명만 들어가는 구간.

### 1-4. 점수와 score_confidence 의 관계

| 상황 | 점수 | score_confidence |
|---|---|---:|
| 사진 양호, 인상 명확 | 정직한 백분위 추정 | 0.75~0.95 |
| 사진 양호, 인상 양가적 | 가운데 값 | 0.55~0.70 |
| 보정 의심 | 보정 제거 추정 | 0.40~0.55 |
| 흐림/조명 불량 | 50 부근으로 모음 | 0.20~0.40 |
| 얼굴 안 보임/단체사진 | 50 (의미 없음) | 0.0~0.20 |

`score_confidence` 가 낮을 때 점수를 극단값(20점 이하 또는 90점 이상)으로 매기는 것은 금지. 신뢰도가 낮을수록 점수는 50 근처로 모은다.

### 1-5. 분포 sanity check

운영 중 다음 분포 가드를 둔다(`python/appearance/eval/distribution_check.py`).

- 누적 100명 이상 분석되었을 때, 점수 분포가 거의 균등(uniform 0~100) 에 가까워야 한다. 단, 서비스 사용자가 자기 선택 편향이 있다면 상위로 살짝 치우치는 정도까지는 허용.
- 평균 점수 ≥ 65 이면 과대평가 편향 의심.
- 60점 이상 비율 ≥ 60% 이면 즉시 회귀 테스트 트리거.
- 30점 미만 비율 < 5% 이면 하위권 회피 편향 의심.

## 2. 여자 벡터 축별 판단 기준

각 축은 0.0 ~ 1.0 사이 실수다. 0.5 가 "평균/중립" 이고, 1.0 에 가까울수록 해당 축이 강하다.

벡터 값은 점수와 다른 차원이다. 외모 절대점수가 낮아도 특정 축은 높을 수 있다(예: 30점이지만 자연스러움 0.8).

### 귀여움
- **고(0.7+)**: 둥근 얼굴/볼, 큰 눈, 동안 인상, 밝은 미소, 작은 체구 느낌
- **중(0.4~0.6)**: 부분적으로 동안 요소 있음, 인상이 한쪽으로 치우치지 않음
- **저(<0.3)**: 성숙하거나 날카로운 인상, 도시적/시크한 분위기

### 청순함
- **고**: 낮은 화장, 자연스러운 헤어, 맑은 피부 인상, 흰/연한 색 옷, 차분한 표정
- **중**: 자연스럽지만 헤어/옷 정돈감이 강한 편
- **저**: 진한 메이크업, 화려한 헤어/옷, 강한 인상

### 시크함
- **고**: 차분/도도한 표정, 어두운 옷, 갸름한 얼굴, 날카로운 눈매
- **중**: 차분하지만 친근함 일부 유지
- **저**: 밝은 미소, 친근한 인상, 부드러운 인상

### 따뜻함
- **고**: 부드러운 인상, 웃는 눈, 친근한 분위기, 둥근 얼굴선, 카디건/니트 같은 포근한 옷
- **중**: 친근하지만 표정이 절제됨
- **저**: 무표정, 차가운 분위기, 도도함

### 스타일리시함
- **고**: 옷-헤어-액세서리 조합의 완성도, 트렌디한 아이템, 정돈된 메이크업
- **중**: 깔끔하지만 패션 신호는 평이
- **저**: 옷이 단순/평범, 패션 신호 거의 없음

### 건강함
- **고**: 생기 있는 피부/표정, 활동적 분위기, 운동복/캐주얼, 밝은 에너지
- **중**: 건강한 인상이지만 활동성 신호는 약함
- **저**: 차분하거나 실내적, 피부 톤이 낮음, 생기 부족

### 성숙함
- **고**: 깊은 인상, 차분한 표정, 분위기 있는 옷, 어른스러운 헤어/메이크업
- **중**: 동안과 성숙의 중간
- **저**: 동안, 어려 보이는 인상

### 지적단정함
- **고**: 깔끔한 헤어, 단정한 셔츠/재킷, 안경, 차분한 표정, 책상/도서관 같은 배경
- **중**: 단정하지만 특별한 지적 신호 없음
- **저**: 캐주얼/스포티 위주, 정돈감 낮음

### 눈큼
- **고**: 실제 눈이 크고 또렷, 시선이 또렷이 느껴짐
- **중**: 보통 크기, 또렷함은 보통
- **저**: 가늘거나 작은 눈, 처진 눈매(작은 인상)

### 부드러운인상
- **고**: 둥근 얼굴선, 처진 눈매, 웃는 눈, 따뜻한 분위기
- **중**: 부드러움과 또렷함의 중간
- **저**: 갸름한 얼굴선, 또렷한 눈매, 차가운 인상

### 날카로운인상
- **고**: 갸름한 얼굴, 선명한 이목구비, 긴/올라간 눈매, 차가운 분위기
- **중**: 부분적으로 선명
- **저**: 둥글고 부드러운 인상

> 부드러운인상 + 날카로운인상의 합이 0.9 ~ 1.1 이 일반적. 같이 1.0 가까이는 거의 없다.

### 자연스러움
- **고**: 보정/화장/연출 흔적 적음, 일상 사진 느낌, 표정 자연
- **중**: 보통 수준의 정돈
- **저**: 과한 보정/필터/메이크업, 매우 연출된 사진

### 화려함
- **고**: 이목구비 선명도 높음, 강한 메이크업, 존재감 있는 스타일, 액세서리
- **중**: 일부 포인트 있음
- **저**: 차분/단정한 분위기, 포인트 없음

## 3. 남자 벡터 축별 판단 기준

### 훈훈함
- **고**: 부드러운 얼굴형, 따뜻한 눈매, 편안한 미소, 친근한 분위기, 니트/캐주얼
- **중**: 부분적으로 친근
- **저**: 무표정/도시적/날카로운 인상

### 댄디함
- **고**: 단정한 셔츠/재킷, 깔끔한 헤어, 정돈된 표정, 도시적 분위기
- **중**: 깔끔하지만 패션 신호는 평이
- **저**: 운동복/후드 위주, 정돈감 낮음

### 시크함
- **고**: 차분/무표정, 어두운 옷, 갸름하거나 날렵한 얼굴, 날카로운 눈매
- **중**: 차분하지만 친근함 일부
- **저**: 밝은 미소, 친근한 인상

### 소년미
- **고**: 작은 얼굴, 동안 인상, 마른 체형, 순한 눈, 덮은 머리/후드
- **중**: 동안 요소 일부
- **저**: 성숙하거나 강한 인상

### 건강함
- **고**: 운동감 있는 체형/턱선, 밝은 피부톤, 활동적 분위기, 스포츠 캐주얼
- **중**: 건강한 인상이지만 운동감 신호는 약함
- **저**: 차분하거나 실내적, 마른/약한 인상

### 지적단정함
- **고**: 안경, 단정한 셔츠, 깔끔한 헤어, 차분한 표정, 도서관/회의실 같은 배경
- **중**: 단정하지만 특별한 지적 신호 없음
- **저**: 운동/스타일/시크 위주

### 남성미
- **고**: 각진 얼굴/턱선, 짙은 눈썹, 운동감 있는 체형, 강한 인상
- **중**: 부분적으로 강함
- **저**: 부드러운 얼굴, 마른 체형, 동안

### 스타일리시함
- **고**: 트렌디한 옷/헤어, 액세서리, 패션 완성도 높음
- **중**: 깔끔하지만 패션 신호 평이
- **저**: 옷이 단순/평범

### 부드러운인상
- **고**: 부드러운 얼굴선, 순한 눈매, 웃는 눈, 따뜻한 분위기
- **중**: 부드러움과 또렷함의 중간
- **저**: 각진 얼굴, 강한 눈매, 차가운 인상

### 날카로운인상
- **고**: 각진/갸름한 얼굴, 선명한 이목구비, 긴/날카로운 눈매, 차가운 분위기
- **중**: 부분적으로 선명
- **저**: 둥글고 부드러운 인상

### 체형탄탄함
- **고**: 운동감 뚜렷, 어깨/팔 라인 명확, 자세 곧음
- **중**: 일반적 체형
- **저**: 마른 체형, 운동감 없음
- 사진이 상반신만이면 판단 가능한 부분까지만 매기고, internal_notes 에 "체형 판단 부분 정보" 표기.

### 자연스러움
- **고**: 보정/화장/연출 흔적 적음, 일상 사진 느낌
- **중**: 보통 수준의 정돈
- **저**: 과한 보정/필터, 매우 연출된 사진

## 4. photo_quality flag 기준

### `single_person`
- true: 화면 안에 명확하게 한 명만, 다른 인물의 얼굴/상반신이 식별 가능한 크기로 보이지 않음
- false: 다른 인물의 얼굴이 식별 가능, 또는 단체사진

### `face_visible`
- true: 얼굴이 화면의 50% 이상 보임, 두 눈 + 코 + 입이 모두 보임
- false: 측면 사진, 마스크/손/머리카락으로 얼굴 절반 이상 가림, 얼굴이 너무 작음

### `lighting_ok`
- true: 자연광 또는 균형 잡힌 실내광, 역광/그림자로 얼굴이 가려지지 않음
- false: 강한 역광, 한쪽이 새까만 그림자, 너무 어두움

### `blurred`
- true: 초점/움직임 블러로 이목구비 판별이 어려움
- false: 이목구비가 또렷이 보임

### `heavy_filter_suspected`
- true: 피부 텍스처가 거의 없음(인형 피부), 눈/턱이 부자연스럽게 변형, AI 필터 흔적
- false: 자연스러운 피부 텍스처와 비례 유지

### `face_occluded`
- true: 마스크, 손, 머리카락, 모자, 안경 반사 등으로 얼굴 30% 이상 가려짐
- false: 얼굴 대부분 노출

### `confidence`
위 6개 flag 의 종합 점수.

| 조건 | confidence |
|---|---:|
| 모든 flag 가 양호 | 0.85~0.95 |
| 한 가지 약한 문제(예: lighting 약간 어두움) | 0.65~0.80 |
| 두 가지 약한 문제 또는 한 가지 강한 문제 | 0.40~0.60 |
| 두 가지 이상 강한 문제 | 0.20~0.40 |
| face_visible=false 또는 single_person=false | 0.0~0.20 |

confidence < 0.5 면 매칭 시스템이 입력으로 받지 않고 사용자에게 사진 재업로드를 요청한다.

## 5. 회귀 테스트 기준

분석기 안정성 검수는 다음 데이터셋과 기준으로 한다.

### 5-1. 데이터셋

`python/appearance/eval/` 아래 6개 폴더에 각 10장씩 동의 받은 샘플 사진을 둔다.

```text
eval_female_low/    여자, 백분위 10~40 (점수 10~40 ground truth)  10장
eval_female_mid/    여자, 백분위 40~70 (점수 40~70)                10장
eval_female_high/   여자, 백분위 70~95 (점수 70~95)                10장
eval_male_low/      남자, 백분위 10~40   10장
eval_male_mid/      남자, 백분위 40~70   10장
eval_male_high/     남자, 백분위 70~95   10장
```

각 폴더에는 `expected.json` 으로 사람이 매긴 정답 점수가 있다. 정답 점수는 사람 평가자 3명 이상의 중앙값.

ground truth 평가자에게도 이 문서의 점수 정의(SCORE_GUIDE.md 백분위 기준)를 먼저 보여줘야 한다. 정의가 어긋나면 검수 자체가 무너진다.

### 5-2. 합격 기준

| 항목 | 기준 |
|---|---|
| 폴더별 평균 점수 | expected 평균 ±5 안 |
| low 폴더 평균 점수 | 50 미만이어야 함(50 이상이면 과대평가 편향) |
| low 폴더 평균 점수 | expected 평균 +5 이내(예: expected 30 → 분석 결과 ≤ 35) |
| mid 폴더 평균 점수 | 40~70 안 |
| high 폴더 평균 점수 | 70 이상이어야 함 |
| 같은 사진 3회 반복 | 점수 표준편차 ≤ 3 |
| 벡터 코사인 유사도(같은 사진 3회) | ≥ 0.95 |
| primary_type 일관성(같은 사진 3회) | 3회 모두 같거나 secondary_types 에 포함 |
| 6 폴더 전체 평균 | 50 ±5 안 (전체 분포가 평균 50 근처여야 함) |

### 5-3. 불합격 시 대응 순서

1. `low` 폴더 평균이 50 이상이면 프롬프트 [하위권 과대평가 절대 금지] 블록을 강화한다.
2. 6 폴더 전체 평균이 55 이상이면 GPT 가 전반적으로 점수를 부풀리는 것. 모델/temperature 조정.
3. 같은 사진 3회 점수 편차가 ±5 초과면 temperature 를 낮추거나 모델 버전을 고정한다.
4. 벡터 코사인 유사도가 0.95 미만이면 프롬프트의 벡터 정의를 더 구체적인 시각 단서 위주로 다시 쓴다.
5. primary_type 일관성이 깨지면 유형 enum 의 정의가 모호하다는 신호. 유형별 판단 기준(섹션 2~3)을 보강한다.

## 6. 매칭에서의 사용

매칭 엔진은 다음 값을 입력으로 받는다.

```python
# 사용자 A (남자)
A_score   = 72    # appearance_score_normalized, 백분위 = 상위 28%
A_conf    = 0.84  # score_confidence
A_vector  = {...}  # 12축
A_pref    = {...}  # preferred_appearance_vector (월드컵 결과, 13축 - 이성)

# 사용자 B (여자)
B_score   = 75
B_conf    = 0.81
B_vector  = {...}  # 13축
B_pref    = {...}  # 12축
```

### 6-1. 외모 궁합 점수

```text
similarity_AB = cosine(A_pref, B_vector)
similarity_BA = cosine(B_pref, A_vector)
score_gap     = |A_score - B_score|   # 백분위 차이

appearance_compatibility =
    0.4 * similarity_AB
  + 0.4 * similarity_BA
  + 0.2 * exp(-score_gap / 15)

# 신뢰도 보정
appearance_compatibility *= min(A_conf, B_conf)
```

`score_gap` 이 백분위 단위라서 의미가 직관적이다. 두 사용자의 점수 차가 15 = 백분위 15% 차이 ≈ exp(-1) = 0.37 가중치. 점수 차가 클수록 양쪽 만족도가 떨어진다는 가정.

### 6-2. 점수대 매칭의 합리성

비슷한 백분위(±10) 의 사용자끼리 매칭되면:

- 양쪽 만족도가 가장 높다.
- 점수 차이가 크면 한쪽이 거절/이탈할 가능성이 커진다.

따라서 점수가 부풀려진 사용자는 결국 그 점수에 맞는 상대를 매칭받게 되고, 만남 후 이탈/노쇼/저평가 리뷰가 발생한다. 이것이 정직한 점수 부여가 사용자 보호와 매칭 품질 모두를 지키는 이유다.

### 6-3. 자기유사 월드컵과의 관계

`/profile/self-worldcup` 에서 사용자가 직접 자기와 비슷하다고 고른 이미지의 점수 평균이 `self_appearance_score` 로 저장된다. 이 값과 GPT 분석 결과 `appearance_score_normalized` 는 같은 백분위 정의를 쓰므로 동일 스케일로 비교 가능하다.

두 값이 크게 다르면(±15 이상):

- 사용자가 자기 외모를 과대/과소 평가한 것이거나
- GPT 분석이 사진 단서를 잘못 해석한 것

운영팀이 검수 후 신뢰도가 더 높은 쪽(보통 사진 여러 장의 분석 평균)을 매칭에 사용. 두 값의 가중 평균을 쓸 수도 있다.

```python
# 가중 평균 권장 식
final_score = (
    self_worldcup_score * 0.4 +     # 사용자 자기 인식
    gpt_score * gpt_confidence * 0.6  # 객관적 분석
) / (0.4 + 0.6 * gpt_confidence)
```

### 6-4. 점수 노출 금지

- `appearance_score_normalized` 를 사용자에게 직접 보여주는 화면은 만들지 않는다.
- 사용자가 자신의 점수 raw 값을 조회하는 API 도 없다.
- 노출 가능한 것은 `primary_type`, `secondary_types`, `visible_features` 까지다.

이 보정 기준을 변경하려면:

6. 이미지 풀(`public/appearance-self/`, `public/appearance-ideal/`)의 ground truth 점수와도 정합성 유지.
