# 매칭 전까지 두근두근 정보 전체 계획

## 한 줄 정의

`두근두근 정보`는 매칭이 잡힌 뒤 실제 만남 전까지, 하루에 한 번 정해진 시간에 카드 한 장을 뽑아 상대에 대한 정보를 조금씩 알아가는 기능이다.

고정된 7일 시간표가 아니라, 남은 기간에 맞춰 카드 후보를 압축하고, 매일 랜덤 뽑기처럼 한 장씩 공개하는 구조로 설계한다.

## 핵심 원칙

- 하루에 한 번만 뽑을 수 있다.
- 정해진 시간대에만 뽑을 수 있다.
- 여러 카드가 뒤집힌 상태로 보이고, 사용자가 한 장을 고르면 그날의 카드가 공개된다.
- 인터넷 논쟁 카드는 전체 기능 중 하나일 뿐이다.
- MBTI, 좋아하는 노래 3곡, 취향, 질문 카드가 함께 섞여야 한다.
- 1:1과 3:3 과팅은 카드 종류와 공개 방식이 다르게 동작해야 한다.
- 만남까지 7일이 안 남았을 때도 자연스럽게 동작해야 한다.

## 기본 사용자 흐름

1. 매칭이 확정된다.
2. 사용자는 `두근두근 정보 카드`에 들어갈 기본 정보를 작성한다.
3. 카드 작성이 끝나면 보증금을 결제한다.
4. 결제가 완료되면 남은 만남일까지 카드 공개 일정이 생성된다.
5. 매일 정해진 시간에 카드 뽑기 화면이 열린다.
6. 사용자는 여러 장의 뒷면 카드 중 하나를 선택한다.
7. 선택한 카드가 뒤집히면서 오늘의 정보가 공개된다.
8. 이미 뽑은 날에는 다시 뽑을 수 없다.

## 카드 뽑기 UX

화면에는 3~5장의 카드가 뒷면으로 나열된다.

예시:

- 카드 A
- 카드 B
- 카드 C
- 카드 D
- 카드 E

사용자가 하나를 누르면 그 카드가 뒤집히고, 오늘 공개될 정보 종류가 나타난다.

예시 결과:

- "오늘은 MBTI 카드가 열렸어요"
- "오늘은 좋아하는 노래 3곡 카드가 열렸어요"
- "오늘은 인터넷 논쟁 카드가 열렸어요"

중요한 점은 사용자가 모든 카드 내용을 고르는 것이 아니라, 오늘 열릴 카드의 연출을 직접 뽑는 느낌을 받게 하는 것이다. 실제 카드 풀과 중복 방지는 서버에서 관리한다.

## 공개 시간 규칙

기본 추천:

- 매일 오후 4시부터 오후 8시까지 뽑기 가능
- 뽑지 않은 경우 그날 기회는 소진됨
- 소진된 카드는 다음 날 자동으로 다시 열리지 않음

최종 정책은 `기회 소진`이다. 카드 뽑기가 매일 들어와서 확인하게 만드는 장치이기 때문에, 놓친 날을 자동 공개하면 긴장감이 약해진다.

소진 규칙:

- 오후 4시부터 오후 8시 사이에 뽑으면 해당 카드 공개
- 오후 8시가 지나도록 뽑지 않으면 `기회 소진` 상태로 기록
- 소진된 카드는 내용이 공개되지 않음
- 이미 뽑았거나 소진된 카드는 다시 뽑기 후보에 들어가지 않음

## 중복 방지 정책

중복 방지는 프론트 연출이 아니라 서버 상태로 막는다.

- 카드 일정은 매칭별, viewer group별, 날짜별로 한 번만 생성한다.
- 카드 종류는 같은 매칭 흐름 안에서 우선순위 배열을 기준으로 한 번씩 배정한다.
- 사용자가 뽑으면 `selected_at`을 기록한다.
- 사용자가 놓치면 `forfeited_at`을 기록한다.
- 다음 카드 후보를 조회할 때 `selected_at` 또는 `forfeited_at`이 있는 카드는 제외한다.
- 따라서 이미 뽑은 카드와 놓친 카드는 다시 뽑히지 않는다.

## 남은 날짜별 운영 방식

7일이 항상 보장되지 않기 때문에, 남은 날짜에 따라 카드 수를 줄인다.

| 만남까지 남은 기간 | 공개 카드 수 | 운영 방식 |
| --- | ---: | --- |
| 7일 이상 | 7장 | 전체 카드 흐름 사용 |
| 5~6일 | 5~6장 | 핵심 카드 우선, 일부 보조 카드 제외 |
| 3~4일 | 3~4장 | MBTI, 노래, 인터넷 논쟁, 만남 전 질문 위주 |
| 1~2일 | 1~2장 | 만남 전 질문, 노래 또는 MBTI만 빠르게 공개 |
| 당일 | 0~1장 | 카드 뽑기보다 만남 안내와 체크인 중심 |

## 카드 우선순위

남은 날짜가 부족할 때는 아래 우선순위로 카드를 남긴다.

### 1:1 우선순위

1. MBTI 카드
2. 좋아하는 노래 3곡 카드
3. 만남 전 질문 카드
4. 인터넷 논쟁 카드
5. 취향/생활 패턴 카드
6. 연애 스타일 카드
7. 기본 분위기 카드

### 3:3 과팅 우선순위

1. 좋아하는 노래 3곡 카드
2. MBTI 카드
3. 인터넷 논쟁 카드
4. 그룹 취향 카드
5. 만남 전 질문 카드
6. 기본 분위기 카드

`연애 스타일 카드`는 1:1 전용으로 둔다. 3:3 과팅에서 연애 성향을 alias별로 공개하면 너무 개인적이고 부담스러울 수 있다.

## 카드 종류 한눈에 보기

| 카드 | 1:1 | 3:3 과팅 | 설명 |
| --- | --- | --- | --- |
| MBTI 카드 | 사용 | 사용 | 상대 성격 유형을 가볍게 공개 |
| 좋아하는 노래 3곡 | 사용 | 사용 | 음악 취향과 감성 공개 |
| 인터넷 논쟁 카드 | 사용 | 사용 | A/B 선택 후 일치 여부 확인 |
| 취향/생활 패턴 카드 | 사용 | 사용 | 주말, 연락, 데이트 스타일 공개 |
| 연애 스타일 카드 | 사용 | 미사용 | 연애관, 갈등 방식 등 1:1 전용 |
| 만남 전 질문 카드 | 사용 | 사용 | 실제 만남에서 쓸 질문을 공개 |
| 기본 분위기 카드 | 선택 | 선택 | 한 줄 소개, 요즘 빠진 것 등 가벼운 정보 |

## 만남 전 질문 카드란?

`만남 전 질문 카드`는 심리 테스트가 아니라, 실제 만남에서 바로 대화로 이어질 질문을 미리 하나 열어주는 카드다.

목적:

- 첫 만남의 어색함 줄이기
- 대화 소재 미리 만들기
- 상대가 어떤 대화를 좋아하는지 파악하기

예시:

- 만나면 제일 먼저 물어보고 싶은 질문은?
- 첫 만남에서 같이 이야기해보고 싶은 주제는?
- 상대가 꼭 알아줬으면 하는 내 취향은?
- 만나면 추천해주고 싶은 음식이나 장소는?
- 오늘 카드 중 제일 기억에 남은 정보는?

1:1에서는 서로의 질문을 직접 공개한다. 3:3에서는 개인 질문이 너무 많아지면 복잡하므로, 그룹 대표 질문 1개 또는 alias별 짧은 질문으로 제한한다.

## 인터넷 논쟁 카드

인터넷 논쟁 카드는 7일 중 하루만 들어가는 재미 카드다.

소스 파일:

`docs/product/matching/internet-debate-question-pool-2026-06-10.md`

사용 방식:

1. 질문 하나를 보여준다.
2. 사용자는 A/B 중 하나를 고른다.
3. 상대 또는 상대 alias와 선택을 비교한다.
4. 일치하면 좋아하는 이모티콘을 보여준다.
5. 불일치하면 아쉬워하는 이모티콘을 보여준다.

예시 질문:

- 탕수육: 부먹 vs 찍먹
- 민초: 가능 vs 불가능
- 짜장 vs 짬뽕
- 물냉 vs 비냉

원본 게시물 문장을 그대로 쓰지 않고, 서비스 톤에 맞게 짧게 다시 쓴다.

## 1:1 공개 방식

1:1에서는 상대와 직접 비교한다.

- MBTI: 나와 상대 MBTI를 함께 표시
- 노래 3곡: 상대의 곡 3개 공개
- 인터넷 논쟁: 내 선택과 상대 선택의 일치 여부 공개
- 취향 카드: 같은 취향이면 작은 일치 표시
- 연애 스타일: 상대의 연애 스타일 답변 공개
- 만남 전 질문: 서로가 작성한 질문 공개

답변형 카드는 둘 다 답변하기 전까지 상대 답변을 먼저 보여주지 않는다.

## 3:3 과팅 공개 방식

3:3에서는 실명 대신 alias를 사용한다.

예시 alias:

- 오소리
- 꿀벌
- 개구리

카드별 처리:

- MBTI: alias별 MBTI 공개
- 노래 3곡: alias별 대표곡 1개 또는 그룹 플레이리스트 공개
- 인터넷 논쟁: 내 선택과 상대 alias별 선택 일치 여부 표시
- 그룹 취향: 상대 그룹의 취향 분포 표시
- 만남 전 질문: 그룹 대표 질문 1개 공개

연애 스타일 카드는 3:3에서 제외한다.

## 카드 배정 로직

서버는 매칭 확정 시점에 아래 정보를 보고 공개할 카드 묶음을 만든다.

- 만남까지 남은 날짜
- 매칭 타입: 1:1 또는 3:3
- 사용자가 작성 완료한 카드 종류
- 이미 공개된 카드
- 필수 카드 우선순위

예시:

만남까지 3일 남은 1:1 매칭이면:

1. MBTI 카드
2. 좋아하는 노래 3곡 카드
3. 만남 전 질문 카드

만남까지 3일 남은 3:3 과팅이면:

1. 좋아하는 노래 3곡 카드
2. 인터넷 논쟁 카드
3. 만남 전 질문 카드

## 데이터 구조 초안

### match_card_pool

매칭별로 공개 가능한 카드 후보를 저장한다.

- `match_id`
- `card_kind`
- `priority`
- `mode`: `one_to_one`, `group`, `both`
- `status`: `available`, `scheduled`, `revealed`, `skipped`

### match_daily_card_schedule

실제 공개 일정을 저장한다.

- `match_id`
- `reveal_date`
- `reveal_window_start`
- `reveal_window_end`
- `selected_card_kind`
- `selected_by_user_id`
- `auto_revealed`
- `revealed_at`

### match_profile_cards

사용자가 미리 작성한 카드 내용을 저장한다.

- `match_id`
- `user_id`
- `card_kind`
- `payload`
- `submitted_at`

### debate_question_pool

인터넷 논쟁 질문 풀이다.

- `id`
- `category`
- `question`
- `option_a`
- `option_b`
- `tone`
- `active`

## 구현 순서

1. 카드 종류와 1:1/3:3 사용 가능 여부를 확정한다.
2. 카드 작성 화면을 만든다.
3. 매칭 확정 후 카드 작성 완료 상태를 확인한다.
4. 보증금 결제 후 남은 날짜 기준으로 카드 풀을 생성한다.
5. 매일 정해진 시간에 카드 뽑기 화면을 연다.
6. 사용자가 고른 카드 또는 자동 공개 카드를 기록한다.
7. 카드 종류별 공개 UI를 구현한다.
8. 3:3 alias 공개 방식을 적용한다.

## 아직 결정할 것

- 하루 카드 뽑기 가능 시간을 몇 시로 할지
- 뽑지 않은 날을 자동 공개할지, 놓친 카드로 둘지
- 카드 뒷면을 3장으로 보여줄지 5장으로 보여줄지
- 3일 이하 매칭에서 인터넷 논쟁 카드를 우선 포함할지
- 3:3 노래 카드를 alias별 대표곡으로 할지 그룹 플레이리스트로 할지

## 추천 결론

두근두근 정보는 `7일 고정 시간표`가 아니라 `남은 날짜에 맞춰 줄어드는 랜덤 카드 뽑기`로 가야 한다. 7일이 남으면 다양한 카드를 충분히 열고, 3일만 남으면 MBTI, 노래, 만남 전 질문처럼 핵심 카드만 압축해서 보여준다. 인터넷 논쟁 카드는 재미 요소로 하루만 넣고, 연애 스타일 카드는 1:1 전용으로 제한한다.
