## 중요한 기준 변경

이전 1차 리뷰에서는 일부 이미지를 "노출/화보 느낌" 때문에 곧바로 재생성 후보로 분류했다.
이 기준은 수정한다.

이상형 월드컵은 사용자의 취향 벡터를 측정하는 도구다.
따라서 건강미, 스타일리시함, 화려함, 성숙함 같은 축을 측정하려면 어느 정도 스타일 차이와 강한 이미지도 필요하다.

새 기준:

```text
노출이 조금 있거나 화보 느낌이 있다는 이유만으로 폐기하지 않는다.
먼저 GPT로 measured_vector를 뽑고,
그 이미지가 어떤 축을 실제로 강하게 대표하는지 확인한 뒤 판단한다.
```

폐기 또는 재생성은 아래 경우에만 한다.

- 선정적이거나 과도하게 성적 대상화된 경우
- 실제 데이팅 앱 프로필 사진으로 보기 어렵게 광고/화보성이 너무 강한 경우
- 같은 얼굴을 옷만 바꾼 것처럼 보이는 경우
- GPT measured_vector 기준으로 어떤 축도 명확히 대표하지 못하는 경우
- 특정 축이 과잉이고 다른 축이 부족해서 풀 균형을 망치는 경우

## 1차 시각 검수 요약

여자 64장 후보셋은 일단 완성되었다.
하지만 최종 확정은 GPT measured 분석 후에 한다.

시각적으로 확인된 특징:

1. `FI01~FI16` 귀여움/청순 구간은 비슷한 한국 AI 프로필 얼굴로 몰리는 경향이 있다.
2. `FI17~FI24` 시크/도도형은 앞 구간과 비교적 잘 갈라진다.
3. `FI25~FI32` 따뜻한/부드러운형은 일부가 스타일/성숙/화려 축으로도 읽힐 수 있다.
4. `FI33~FI40` 스타일리시/화려형은 축 신호가 강하다. 폐기보다 벡터 분석 우선.
5. `FI41~FI48` 건강/활동형은 건강미 축 측정에 유용할 수 있다. 노출만으로 폐기하지 않는다.
6. `FI49~FI56` 성숙/분위기형은 성숙/화려/스타일리시 축으로 강하게 읽힐 수 있다.
7. `FI57~FI64` 지적/단정형은 대체로 단정하지만 일부 얼굴 공식 유사성이 있다.

## 지금 당장 재생성하지 않는 이유

현재 필요한 것은 "예쁜 사진을 고르는 감"이 아니라, 각 사진이 실제로 어떤 벡터값을 갖는지 확인하는 것이다.

예를 들어 `FI47`이 건강형으로 강하게 읽힌다면, 의상 노출이 조금 있어도 건강/활동형 선호 측정에 유용할 수 있다.
반대로 `FI06`이 target은 귀여움이어도 GPT가 청순/성숙/지적으로 읽는다면 귀여움 슬롯으로 쓰면 안 된다.

따라서 다음 단계는 재생성이 아니라 GPT 재분석이다.

## 다음 작업

1. `FI01~FI64` 전체를 GPT 분석 프롬프트로 재평가한다.
2. 결과를 `ANALYSIS_RAW_FEMALE.json`에 저장한다.
3. 각 이미지의 `measured.appearance_vector`를 기준으로 bucket score를 계산한다.
4. `final_bucket`을 measured 기준으로 재배치한다.
5. 축별 분포를 계산한다.
6. 부족한 축/유형을 찾는다.
7. 부족한 유형만 Manus에게 추가 생성 요청한다.
8. 과잉 유형은 active에서 제외하거나 candidate로 내린다.
9. 최종 `METADATA_FEMALE.json`을 확정한다.

## measured 분석 후 판단할 항목

각 이미지별로 아래를 판단한다.

| 항목 | 설명 |
|---|---|
| measured_score | 실제 외모 점수대가 목표 분포와 맞는가 |
| primary_type | target_type과 일치하는가, 아니면 다른 유형으로 재배치해야 하는가 |
| appearance_vector | 어떤 축을 강하게 대표하는가 |
| bucket_scores | 8개 여자 유형 중 어느 버킷에 배치할 수 있는가 |
| bucket_gap | 한 유형이 명확한가, 혼합형인가 |
| duplicate_risk | 같은 얼굴 공식으로 보이는가 |
| usefulness | 최종 월드컵에서 취향 측정에 유용한가 |

## 부족 유형 판단 방식

최종 목표는 target 기준 8장씩이 아니다.
최종 목표는 measured/final_bucket 기준으로 8유형 균형을 맞추는 것이다.

여자 final_bucket 목표:

| final_bucket | 목표 수량 |
|---|---:|
| 귀여운/동안형 | 8 |
| 청순/자연형 | 8 |
| 시크/도도형 | 8 |
| 따뜻한/부드러운형 | 8 |
| 스타일리시/화려형 | 8 |
| 건강/활동형 | 8 |
| 성숙/분위기형 | 8 |
| 지적/단정형 | 8 |

예시:

```text
GPT 분석 결과:
청순/자연형 18장
귀여운/동안형 4장
건강/활동형 5장
스타일리시/화려형 11장

판단:
- 청순은 과잉이므로 상위 8장만 active 후보.
- 귀여움은 4장 부족하므로 Manus에게 귀여움 후보 추가 생성 요청.
- 건강은 3장 부족하므로 건강 후보 추가 생성 요청.
- 스타일리시는 8장만 active, 나머지는 candidate.
```

## Manus에게 추가 작업을 맡기는 방식

Manus에게 바로 "문제 있어 보이는 사진을 다시 만들어라"라고 시키지 않는다.
먼저 GPT measured 분석 결과를 보고 부족한 축을 특정한 뒤 요청한다.

```text

아래 조건을 만족하는 후보 이미지를 N장 추가 생성해라.

- target bucket:
- 부족한 vector 축:
- 피해야 할 과잉 축:
- 기존 유사 이미지:
- 반드시 다르게 만들 얼굴/헤어/옷/배경:
```
